# Security Policy

ModelFuzz is a security tool, so we take vulnerabilities in it seriously.

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Instead, email **gagan.ping@gmail.com** with:

- A description of the vulnerability and its impact
- Steps to reproduce (a minimal repro is ideal)
- Affected version(s)

We aim to acknowledge reports within 48 hours and to ship a fix or mitigation
as quickly as possible depending on severity.

## Supported Versions

Only the latest published version on [PyPI](https://pypi.org/project/modelfuzz/)
is supported. Please upgrade before reporting an issue that may already be fixed.
